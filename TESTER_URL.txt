## Step-by-Step Quick Ref

### 1️⃣ Backend Deployed First
- Render creates: `https://messenger-backend-XXXXX.onrender.com`
- Neon provides: `postgresql://...@ep-XXXXX.aws.neon.tech/...`

### 2️⃣ Frontend Deployed Second
- Build command includes: `--dart-define=BACKEND_URL=https://messenger-backend-XXXXX.onrender.com`
- Render hosts: `https://messenger-frontend-XXXXX.onrender.com`

### 3️⃣ Tester Opens
- Any browser → `https://messenger-frontend-XXXXX.onrender.com`
- ✅ App loads
- ✅ Connects to your backend
- ✅ Uses Neon database
- ✅ Ready to test!

## Common Questions

**Q: Does the tester need to install anything?**  
A: No! Just a web browser.

**Q: Will it work from their phone?**  
A: Yes! Use the same URL on phone browser.

**Q: How is the backend URL configured?**  
A: It's embedded during build time via `--dart-define=BACKEND_URL=...`

**Q: What if the tester goes offline?**  
A: The app needs internet. It won't work completely offline.

**Q: Can multiple testers use it at same time?**  
A: Yes! Render handles concurrent requests.

**Q: How long can they test?**  
A: As long as you keep both services running on Render.
